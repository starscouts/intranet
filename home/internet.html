Internet Access

If you connect with Wi-Fi, you have Internet access through Equestria.dev's routing infrastructure. Note that, while the servers are in France, the exit node router is not in France. We are currently using routers in Zurich, Switzerland.

Known public IP addresses

Here is a list of all the public addresses that are known to be used by the routers:

  • 193.32.127.66
  • 193.32.127.67
  • 193.32.127.68
  • 193.32.127.69
  • 193.32.127.70

Note that rotations happen randomly, so you should not rely on the public IP address staying the same. It is also possible for other people to have the same IP address, therefore it is best not to trust it for authentication. If you think we should add more servers (maybe other locations), please get in touch with your administrator.

Physical link

The exit node routers are linked to the intranet router using a WireGuard tunnel for optimized performance. IPv6 is disabled on the tunnel side, but the router should support it if we ever decide to turn it on. The internet access itself is provided by 31173 on servers owned by Mullvad.

Why Switzerland?

Equestria.dev is French, so choosing Switzerland for the Internet location might not make a lot of sense. However, here are some reasons why we chose Switzerland over another country (such as France):

  • Switzerland's commitment to privacy means that government spying will be made harder (or impossible). Additionally, no data retention is required, which allows service providers to store no logs.
  • Political neutrality is an important subject for a service provider. Since Switzerland is neutral, censorship should not happen, leaving the internet free and open for everyone.
  • The internet infrastructure in Switzerland is well developped, meaning more reliability and faster speeds, even through a lot of different routers.
